# 收息佬前端状态转换与导航约束

## 1. 目标

本文档定义前端页面的状态机、按钮动作、前置条件与失败态，确保：

1. 每个可见按钮都有明确行为
2. 页面之间跳转可回溯、可恢复上下文
3. 数据状态（loading/error/empty/no-data/ready）一致
4. 不再展示“静态假数据冒充真实结果”

## 2. 术语

- `symbol`: A 股 6 位代码（`^(6|0|3)\d{5}$`）
- `symbols`: 对比页代码列表（去重、逗号分隔）
- `portfolio`: 本地投资组合持仓（name 必填，symbol 可选，股数、成本价必填）

## 3. 全局状态机

每个页面都必须只处于以下状态之一：

1. `loading`: 正在请求/计算
2. `error`: 请求失败或参数非法
3. `empty`: 用户尚未完成必要输入（如未选标的）
4. `no-data`: 输入有效但无可展示结果
5. `ready`: 数据可展示并可交互

状态优先级：

1. `loading`
2. `error`
3. `empty`
4. `no-data`
5. `ready`

## 4. 页面状态与转换

## 4.1 投资组合页（`/`）

### 核心状态

1. `portfolio-empty`: 无持仓
2. `portfolio-ready`: 有持仓
3. `quote-loading`: 正在刷新估值
4. `quote-partial-error`: 部分标的估值失败

### 关键动作

1. `导出报告`
   - 前置：`portfolio-ready`
   - 动作：导出当前持仓估值与收益 CSV
   - 失败：无持仓时禁用
2. `添加资产`
   - 前置：无
   - 动作：打开添加资产面板，支持搜索 + 录入股数/成本
   - 约束：代码为可选；若填写代码必须为 A 股 6 位
3. `刷新估值`
   - 前置：`portfolio-ready`
   - 动作：按持仓逐个获取最新详情并更新估值
4. `编辑/删除持仓`
   - 前置：行存在
   - 动作：修改股数成本或删除持仓
5. `查看详情`
   - 前置：行 symbol 有效（无代码资产仅保留组合统计，不支持详情）
   - 动作：跳转 `/stock-detail?symbol=...`

## 4.2 股息详情页（`/stock-detail`）

### 核心状态

1. `empty`: 无 symbol
2. `no-data`: 基础信息或分红历史缺失
3. `ready`: 可展示历史、估算、明细

### 关键动作

1. `进入回测`
   - 前置：`ready`
   - 动作：跳转 `/backtest?symbol=...`
2. `记录最近浏览`
   - 前置：symbol 有效
   - 动作：更新最近浏览队列

## 4.3 自选页（`/watchlist`）

### 核心状态

1. `empty`: 无自选数据
2. `ready`: 有自选数据

### 关键动作

1. `查看首个详情`
   - 前置：至少 1 条
2. `对比前 3 只`
   - 前置：至少 2 条

## 4.4 对比页（`/comparison`）

### 核心状态

1. `empty`: 无 symbols 输入
2. `no-data`: 请求成功但结果为空
3. `ready`: 展示对比表

### 关键动作

1. `查看 xxx 详情`
   - 前置：行 symbol 有效
   - 动作：跳转详情并保留 compare 上下文

## 4.5 回测页（`/backtest`）

### 核心状态

1. `empty`: 无 symbol
2. `no-data`: 有结果对象但无交易流水
3. `ready`: 有摘要和流水

## 5. 路由与上下文恢复规则

1. 详情页 symbol 解析顺序：`query.symbol > path.symbol > rememberedLastSymbol > 默认值`
2. 对比页 symbols 解析顺序：`query.symbols > path.symbols > rememberedComparisonSymbols > 默认值`
3. 最近浏览队列最多保留 10 条，重复访问置顶
4. 所有跨页跳转都应写入必要上下文（last symbol / compare symbols）

## 6. 真数据约束

1. 投资组合汇总指标必须来自本地持仓 + 实时详情计算
2. 机会列表必须来自真实代码集合，不允许写死美股演示文案
3. 最近浏览必须来自真实访问记录
4. 快捷工具按钮必须可跳转，禁止无动作按钮

## 7. 验收清单

1. 投资组合页所有主按钮均有明确动作并可复现
2. 可完成“搜索股票（代码或名称） -> 添加持仓（代码可选） -> 刷新估值 -> 查看详情 -> 进入回测”
3. 投资组合汇总数值随持仓变化实时更新
4. 在空持仓与有持仓两种情况下，状态表达与禁用逻辑正确
5. 主路径无假数据文案，全部来自真实 symbol 或本地真实持仓
