# DividendMonitor PRD

## 1. 文档信息

- 产品名称：DividendMonitor
- 产品定位：面向长期投资者的攒股收息桌面分析工具
- 终端形态：Windows 桌面端
- 覆盖市场：A 股（股票 / ETF / 场外基金）
- 数据策略：优先使用免费公开接口
- 当前阶段：V1 核心功能已交付，多资产架构已落地
- 文档目标：明确首期可交付范围、用户价值、功能边界和分阶段路线图

## 2. 产品背景

面向偏长期持有、重视股息现金流的投资者，现有工具通常存在以下问题：

- 偏交易导向，缺少围绕股息率、分红稳定性、分红覆盖能力的长期分析视角
- 股息率口径混乱，难以按自然年、指定年份或年份区间统一比较
- 难以基于去年的分红金额、分红比例和总股本结构，对未来股息率进行快速估算
- 多股票横向比较零散，无法沉淀为个人自选池和可视化地图
- 很多工具依赖登录、云同步或在线账户体系，不适合先快速验证桌面端产品价值

因此需要一个第一阶段专注 Windows 本地使用、免登录、免云端、强调股息分析与对比能力的桌面应用。

## 3. 产品目标

### 3.1 核心目标

1. 帮助用户快速分析股票在不同年份和区间下的历史股息率
2. 帮助用户基于公开财务与分红数据，估算未来股息率
3. 帮助用户建立自选股观察池并进行多维比较
4. 帮助用户通过图表和地图更直观地理解股息机会分布
5. 用分阶段方式验证产品可行性，为后续加入更多资产比较场景打基础

### 3.2 非目标

- 首期不做账号体系
- 首期不做云同步
- 首期不做交易下单
- 首期不做社区、资讯、评论
- 首期不覆盖移动端
- 首期不做房价租售比功能，仅在路线图中预留

## 4. 目标用户

### 4.1 核心用户

- 长期持股收息投资者
- 关注分红稳定性和现金流回报的价值投资者
- 喜欢维护股票观察池并做横向比较的个人投资者

### 4.2 次级用户

- 想从“股息率”视角建立投资框架的新手用户
- 想把股票与房产租售比放在同一收益框架中比较的资产配置用户

## 5. 用户痛点

1. 查股息率时，不同平台算法不一致，难以统一决策口径
2. 看一年分红容易，但看多年区间趋势和稳定性很麻烦
3. 想估算未来股息率时，需要手工拼凑分红比例、利润、总股本等多个维度
4. 比较多只股票时，数据维度多、切换页面频繁、效率低
5. 缺少直观可视化，难以快速发现高股息标的或板块分布特征

## 6. 产品定位

DividendMonitor 是一款“以股息分析为中心”的本地桌面应用，不追求大而全的行情终端，而是围绕“收息决策”提供统一口径计算、估算、比较与可视化能力。

## 7. 使用场景

### 7.1 场景一：复盘自然年股息率

用户希望查看某只股票在 2020-2025 每个自然年的股息率变化，用于判断分红稳定性。

### 7.2 场景二：估算未来股息率

用户看到公司去年分红总金额和分红比例后，希望结合当前市值或股价快速估算未来股息率是否有吸引力。

### 7.3 场景三：自选池筛选

用户维护一个自选股列表，希望从中筛出股息率高、利润稳定、市盈率合理的标的。

### 7.4 场景四：横向比较

用户想比较 3-10 只股票的市值、利润、ROI、市盈率、历史股息率、估算股息率等指标。

### 7.5 场景五：地图洞察

用户希望通过股息率地图快速理解行业、市场或自定义分组中的收益分布情况。

## 9. 关键需求明细

### 9.1 股票搜索与选择

- 支持按股票代码、名称搜索
- 支持从搜索结果进入详情分析
- 支持加入或移出自选列表

### 9.2 自选股管理

- 支持查看自选股票列表
- 支持本地保存
- 支持按股息率、市值、市盈率等字段排序
- 支持后续扩展自定义分组

### 9.3 自然年股息率计算

- 支持按自然年统计某只股票当年的分红事件与年度累计股息率
- 支持展示计算公式和数据来源字段
- 支持指定单年查看
- 支持按年份区间查看趋势
- 支持展示年度内分红事件明细
- 计算时需考虑总股本变动对单次分红口径的影响

建议统一口径如下：

- 首期采用“事件级股息率累加口径”，即按每次现金分红分别计算当次股息率，再按自然年汇总
- 单次分红股息率 = 该次每股现金分红 / 股权登记日前一交易日收盘价
- 自然年股息率 = 该自然年内所有现金分红事件的单次股息率之和
- 自然年归属默认按除权除息日所属自然年统计

选择该方案的原因：

1. 属于市场中已有且较常见的事件口径，便于和公开资料互相校验
2. 对定增、配股、回购注销等股本变化更稳健，因为每次分红都使用当次事件对应的股份口径和价格环境
3. 避免用“全年累计每股分红 / 年末价”这类方式造成跨期股本变化后的失真
4. 更符合“投资者在分红发生前持有股票所获得的现金回报率”这一分析目标

### 9.4 未来收益率估算

股票和 ETF/基金使用不同的估算策略：

**股票方法**（`estimateFutureYield`）：
- 基准估算：沿用上一年度分红比例 × 最新年度净利润 → 预估总分红 → 每股分红 → 股息率
- 保守估算：延续上一年度分红总额 → 每股分红 → 股息率
- 输入：`latestPrice`、`latestTotalShares`、`latestAnnualNetProfit`、`lastAnnualPayoutRatio`、`lastYearTotalDividendAmount`

**ETF / 基金方法**（`estimateFundFutureYield`）：
- 基准估算：最近完整年份的每股分配合计 ÷ 最新价格
- 保守估算：最近 1-3 年平均年分配 ÷ 最新价格
- 输入：`latestPrice`（ETF 用市价，场外基金用单位净值）、`dividendEvents`（历史分配事件）

展示要求：

1. 必须展示采用的是哪一种估算方法
2. 必须展示完整计算链路
3. 若同时给出两种估算结果，应明确标注”基准”与”保守”
4. 前端根据估算来源区分展示：股票显示财务输入（净利润/分红率/总股本），基金显示分配输入（可用年份/年均分配）

### 9.5 多资产比较

支持股票、ETF、场外基金横向展示：

**通用指标**：
- 最新价、名称、代码、资产类型

**股票专属**：
- 总市值、市盈率、市净率、平均股息率、未来股息率估算
- ROI 或 ROE/ROIC 类收益指标
- 市盈率
- 历史自然年股息率
- 区间平均股息率
- 估算未来股息率
- 分红比例
- 总股本

### 9.6 股息复投回测

- 用户可选择一个历史买入时间点
- 默认一次性买入后长期持有，不做主动加仓减仓
- 每次现金分红到账后，自动按设定规则复投同一股票
- 输出区间总收益率、年化收益率、期末持仓股数、累计分红金额和复投次数
- 支持查看回测过程明细

首期建议规则：

1. 买入价格采用所选交易日收盘价
2. 分红归属采用派息到账日
3. 复投价格采用派息到账日后最近一个交易日收盘价
4. 若存在送转增，仅调整持仓股数，不计入现金分红收益
5. 年化收益率采用 CAGR 口径

### 9.7 股息率地图

- 支持以热力或矩形树图方式展示股票股息率分布
- 支持按行业、板块、自选分组等维度展示
- 支持颜色映射和图例
- 支持点击地图项进入股票详情或比较视图

### 11.2 深度分析

3. 切换年份或年份范围

5. 进入回测页验证长期持有复投效果
6. 打开地图页观察分布

## 14. 非功能需求

1. 支持 Windows 10/11
2. 首次打开在可接受时间内完成初始化
3. 数据计算需可解释、可追溯
4. 本地数据保存可靠，异常退出后尽量不丢失配置
5. UI 简洁、信息密度适中，适配桌面宽屏

## 15. 风险与约束

1. 免费 A 股公开接口的稳定性和限流策略可能变化
2. 分红公告、除权、到账时间可能跨年，需严格统一自然年归属规则
3. 回测涉及送转增、停牌、现金到账时点等细节，必须在界面中说明假设
4. ROI 指标定义可能存在歧义，首期需明确定义替代指标
5. 无云端架构意味着用户数据默认只保存在本地

1. 用户可搜索并保存自选股票
2. 用户可查看至少一只股票的自然年股息率
3. 用户可选择单年或年份范围
