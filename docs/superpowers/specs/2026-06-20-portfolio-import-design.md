## 1. 初始诉求

Alex 希望能从支付宝理财通 / 腾讯理财通直接导入资产持仓信息，免去逐笔手动录入。

## 2. 可行性调研结论

### 2.1 官方 API 直连

支付宝与理财通**均未**向个人开发者开放"代用户查询理财持仓"的官方 API。账户隐私受严格保护，此路径不可行。

### 2.2 截图 OCR

技术可行但质量不稳，且需引入 OCR 依赖（Tesseract.js 或云端 OCR），开发量大、用户体验差。**决定不做。**

### 2.3 理财通文件导出

调研发现腾讯理财通**没有持仓导出入口**。此来源无法通过文件导入实现。**砍掉。**

### 2.4 支付宝文件导出

支付宝理财持仓**只能导出 PDF 格式的"资产证明"**，没有 CSV/Excel 选项。

## 3. 支付宝 PDF 格式分析

### 3.1 样本

- 样本文件：`pdf/陈鼎亚_20260620132042473_2088142637078413.pdf`
- 解析结果：`pdf/extracted.json`
- 解析工具：`pdfplumber`（Python，已验证可用）

### 3.2 PDF 结构

**第 1 页 — 封面**：
- 编号（如 `20260620132042473_2088142637078413`）
- 代理机构：蚂蚁（杭州）基金销售有限公司
- 客户姓名、身份证号
- 截止日期（如 `2026-06-20`）→ 可作为整体快照日
- 资产总额（如 `3979.27元`）

**第 2 页 — 持仓明细表**，8 列：

| 列名 | 示例 | 映射 |
|------|------|------|
| 序号 | `1` | 行号 |
| 基金交易账号 | `41023230720261524` | — |
| 基金名称 | `博时黄金ETF联接C` | `name` |
| 基金代码 | `002611` | `code` → 直接构造 `AssetKey` |
| 总份额 | `11.58` | `shares` |
| 单位净值 | `2.943` | 当前净值（非平均成本） |
| 单位净值日期 | `20260618` | `snapshotDate`（`YYYYMMDD` → `YYYY-MM-DD`） |
| 资产小计 | `34.08` | = 份额 × 净值 |

### 3.3 格式注意事项

- **有基金代码**：可直接构造 `AssetKey`（`FUND:A_SHARE:002611`），不需要名称模糊匹配
- **名称有换行符**：如"华夏国证自由现金流ETF联\n接A"，需清理
- **日期格式**：`YYYYMMDD`，需转 `YYYY-MM-DD`
- **无平均成本**：只有"当前单位净值"，与现有 `PortfolioPositionDto.avgCost`（平均买入成本）语义不同

## 4. 搁置原因

调研完成后发现以下问题使成本收益比不划算：

1. **avgCost 语义不匹配**：支付宝 PDF 只有"当前单位净值"没有"平均买入成本"。用净值近似 avgCost 会导致收益率显示为 ~0%，失去导入意义；新增 `currentNav` 字段则需改 schema + 改 UI + 改收益率计算逻辑，工作量大
2. **理财通无法导出**：只支持支付宝一个来源，覆盖面窄
3. **PDF 解析引入重依赖**：Electron 端需要 `pdfplumber`/`pdfjs-dist` 等 PDF 解析库，体积不小且中文字体处理有坑
4. **格式不稳定风险**：支付宝随时可能改 PDF 格式，维护成本高

综合评估后 Alex 决定**暂不实现此功能**，保留调研记录供后续参考。

## 5. 如果将来要实现的技术路线

若未来支付宝开放更好的导出格式，或社区出现成熟的 PDF 解析方案，可参考以下路线：

1. **schema**：`portfolio_positions` 加 `source`（`MANUAL`/`ALIPAY`）+ `snapshotDate` 列
2. **解析层**：`src/main/application/importers/alipayPdfImporter.ts`，用 `pdfjs-dist` 在 Electron 主进程解析 PDF 表格
3. **用例**：`parseImportFile`（PDF → 预览行）+ `commitImportedPositions`（预览 → 写库），拆两次 IPC
4. **前端**：Dashboard 加"导入持仓"按钮 + Modal（Upload → 预览表 → 确认）
5. **avgCost 处理**：新增 `currentNav` 字段与 `avgCost` 并存，导入只填 `currentNav`，`avgCost` 留空让用户手动补
6. **冲突策略**：同源同 assetKey 覆盖、异源新增、快照日期更早默认跳过
