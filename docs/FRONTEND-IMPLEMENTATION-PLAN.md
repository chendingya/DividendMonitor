# 收息佬前端实现说明

## 1. 文档目标

本文档记录当前前端已经落地的页面、主路径、状态约束和持续演进项，用于回答三类问题：

1. 当前页面与运行时链路到底长什么样
2. 前端主路径如何在 Electron 与浏览器预览两种运行时下保持一致
3. 后续还要补什么，但哪些事情当前还没有做

## 2. 当前现状

### 2.1 已完成

1. Electron + React + IPC 主链路已打通
2. 目录结构已完成第一轮简化
3. 概览页、个股详情页、自选页、对比页、回测页均已落地
4. 桌面端已接入真实数据链路，浏览器预览端默认改为通过本地 HTTP API 走同一套真实用例
5. 个股详情页与对比页已支持 `PE/PB` 和 `10Y/20Y` 估值分位展示
6. 自选页已支持批量对比、已选上下文记忆和返回链路恢复

### 2.2 主要缺口

1. 双端虽然已统一到同一业务数据源，但仍通过 IPC 与 HTTP 两条传输通道接入
2. mock runtime 仍保留为显式调试兜底，需要继续控制其使用边界
3. SQLite 当前已覆盖自选与持仓持久化，但完整缓存层尚未形成
4. `yield-map`、`settings` 仍是预留能力，当前尚未落地页面
5. 空态/错误态/无数据态已部分统一，但还未提炼为更完整的通用组件层

## 3. 页面与路径

### 3.1 当前页面

| 路由 | 页面 | 当前用途 |
| --- | --- | --- |
| `/` | `DashboardPage` | 搜索股票、录入持仓、跳转详情、加入自选 |
| `/stock-detail` | `StockDetailPage` | 历史股息率、未来股息率估算、PE/PB 估值分位、自选操作 |
| `/watchlist` | `WatchlistPage` | 自选列表、排序筛选、批量选择、进入对比 |
| `/comparison` | `ComparisonPage` | 多股指标对比、行内进入详情、10Y/20Y 估值窗口切换 |
| `/backtest` | `BacktestPage` | 股息复投回测、交易流水与假设说明 |

### 3.2 当前主路径

1. 搜索 -> 详情
2. 搜索 -> 加入自选 -> 自选页
3. 自选页 -> 对比页 -> 详情页
4. 详情页 -> 回测页
5. 返回自选页后恢复上一次选中的对比标的

## 4. 设计与实现原则

1. 先主路径闭环，再补外围页面
2. 先交互一致性，再做视觉精修
3. 计算口径以主进程服务返回为准，前端负责解释与呈现
4. 页面只依赖 renderer service，不直连 Node 能力
5. 浏览器预览允许通过独立运行时接入后端，但不应再退化为默认 mock 数据链路

## 5. 运行时路径

### 5.1 Electron 桌面端

`page -> hook -> renderer service -> desktopApi -> preload -> IPC -> use case -> repository -> adapter`

### 5.2 浏览器预览

`page -> hook -> renderer service -> desktopApi -> browserHttpRuntimeApi -> local HTTP API -> main use case`

### 5.3 一致性约束

1. 页面层不出现 Electron/Web 双分支散落逻辑
2. 所有跨页跳转都通过统一路由工具写入必要上下文
3. 桌面端和浏览器预览共用同一套页面、hook 和组件

## 6. 状态与路由约束

### 6.1 页面状态

每个页面只应处于以下状态之一：

1. `loading`
2. `error`
3. `empty`
4. `no-data`
5. `ready`

### 6.2 路由上下文恢复

1. 详情页支持从 query、path 和最近浏览 symbol 恢复
2. 对比页支持从 query、path 和最近对比 symbols 恢复
3. 自选页支持记忆最近一次选中的对比标的
4. 批量选择返回时不应丢失上下文

## 7. 当前已落地能力

1. 桌面端与浏览器预览已共享真实搜索、详情、对比、回测数据链路
2. 自选与工作台持仓都已切到 SQLite 持久化
3. 详情页已支持年度股息率、未来股息率估算、PE/PB 估值分位
4. 对比页已支持排序、高亮、行内详情入口和估值窗口切换
5. 回测页已支持选日期、查看摘要和交易流水
6. UI 已统一到 `theme.css` 中的 `ledger-*` 风格体系

## 8. 当前未完成项

1. 本地 HTTP API 的部署/启动说明还需要继续完善
2. SQLite 还没有扩展为完整缓存层
3. 地图页与设置页仍未落地
4. 更通用的页面状态组件仍可继续抽取
5. 估值链路尚未升级到 SQLite 缓存或多数据源切换

## 9. 验收建议

建议按以下顺序验收：

1. 桌面端：搜索 -> 加入自选 -> 自选 -> 对比 -> 详情 -> 回测
2. 浏览器预览：搜索 -> 自选 -> 对比 -> 详情 -> 回测
3. 详情页：检查历史股息率、未来股息率估算、`10Y/20Y` 估值分位
4. 对比页：检查列排序、行内详情入口、估值窗口切换
5. 重启桌面端：确认自选、持仓和 SQLite 文件保留
