# 收息佬手动验证手册

## 1. 文档目标

本文档用于指导你在当前仓库状态下手动验证：

1. Electron 桌面端主链路是否正常
2. 浏览器预览 fallback 是否正常
3. 自选 SQLite 持久化是否正常
4. 当前已知限制是否与文档一致

本文档按“操作步骤 + 预期结果 + 失败判定”组织。

## 2. 验证前准备

## 2.1 环境要求

1. Windows
2. 已安装项目依赖
3. 可在项目根目录运行 `npm run dev`

## 2.2 启动方式

在项目根目录执行：

```bash
npm run dev
```

预期结果：

1. Electron 开发窗口启动
2. 渲染层预览地址可见，默认应为 `http://127.0.0.1:8192/`

失败判定：

1. Electron 窗口未打开
2. 终端出现构建失败
3. 页面白屏且无法恢复

## 2.3 当前开发态数据目录

开发态数据目录当前位于：

- `i:\code\DividendMonitor\.runtime-data`

重点关注：

- `i:\code\DividendMonitor\.runtime-data\db\dividend-monitor.sqlite`

说明：

1. 这是开发态 Electron 的 `userData` 目录
2. 当前桌面端自选 SQLite 文件会写到这里

## 3. 验证范围

本轮建议验证两条路径：

1. `Path A`：Electron 桌面端真实链路
2. `Path B`：浏览器预览 fallback 链路

## 4. Path A：Electron 桌面端真实链路

## 4.1 目标

验证以下能力：

1. 搜索
2. 加入自选
3. 自选页展示
4. 批量进入对比
5. 详情页展示
6. 回测页展示
7. 自选重启后保留

## 4.2 步骤 1：桌面端搜索股票

操作：

1. 打开 Electron 窗口首页
2. 在“输入股票代码或名称”框中输入 `600519`
3. 点击“搜索”

预期结果：

1. 页面出现搜索结果卡片
2. 卡片中至少有：
   - `录入持仓`
   - `查看详情`
   - `加入自选`

失败判定：

1. 搜索后无结果且无错误提示
2. 页面报错
3. 控件无法点击

## 4.3 步骤 2：加入自选

操作：

1. 在搜索结果中点击“加入自选”

预期结果：

1. 页面无崩溃
2. 无 `EPERM` 写入错误弹窗
3. 如页面有消息提示，应提示加入成功或无失败提示

失败判定：

1. 出现 `Error invoking remote method 'watchlist:add'`
2. 出现 `EPERM`
3. 页面卡死或崩溃

## 4.4 步骤 3：检查 SQLite 文件是否存在

操作：

1. 打开目录 `i:\code\DividendMonitor\.runtime-data\db`

预期结果：

1. 存在 `dividend-monitor.sqlite`

失败判定：

1. 执行过“加入自选”后仍无数据库文件

## 4.5 步骤 4：进入收益追踪页

操作：

1. 点击左侧导航“收益追踪”

预期结果：

1. 进入自选页
2. 页面能看到刚才加入的股票
3. 显示基础字段：
   - 股票代码
   - 名称
   - 最新价
   - 估算未来股息率

失败判定：

1. 列表为空但之前已加入成功
2. 页面报错
3. 字段全部为空且无说明

## 4.6 步骤 5：再加入一只股票

建议测试代码：

- `000651`

操作：

1. 回到首页
2. 搜索 `000651`
3. 点击“加入自选”
4. 回到收益追踪页

预期结果：

1. 自选页出现两只股票
2. 自选数量统计更新
3. 最高估算股息率、平均未来股息率等摘要更新

失败判定：

1. 第二只股票无法加入
2. 加入后列表未刷新
3. 数据重复或顺序异常且无规律

## 4.7 步骤 6：批量选择并进入对比

操作：

1. 在收益追踪页分别点击两行的“选择”
2. 确认“已选 2 只”
3. 点击“批量进入对比”

预期结果：

1. 进入对比页
2. URL 中带有 `symbols=...`
3. 表格展示两只股票
4. 表格至少包含：
   - 最新价
   - 市盈率
   - 市净率
   - 区间平均股息率
   - 估算未来股息率
   - PE/PB 分位列

失败判定：

1. 按钮无法点击
2. 跳转后空白
3. 只展示部分股票

## 4.8 步骤 7：从对比页进入详情页

操作：

1. 在对比页点击“查看 600519 详情”或任一详情按钮

预期结果：

1. 进入详情页
2. 可看到：
   - 基础信息
   - 年度股息率图
   - 未来股息率估算区
   - PE/PB 估值卡片
   - `10Y / 20Y` 估值窗口切换
   - 加入/移出自选按钮

失败判定：

1. 跳转失败
2. 页面错误态无说明
3. 图表、估值区和估算区都为空且无解释

## 4.9 步骤 8：从详情页进入回测页

操作：

1. 在详情页点击回测入口
2. 如可选日期，选择默认日期或保留默认值

预期结果：

1. 进入回测页
2. 页面至少展示：
   - 结果摘要
   - 假设说明
   - 分红与复投流水

失败判定：

1. 页面报错
2. 无结果且无原因说明
3. 假设说明缺失

## 4.10 步骤 9：验证自选持久化

操作：

1. 关闭 Electron 开发窗口
2. 保持 `npm run dev` 重新启动或重新运行
3. 再次打开收益追踪页

预期结果：

1. 之前加入的自选仍然存在
2. 无需重新加入

失败判定：

1. 列表被清空
2. SQLite 文件存在但页面为空

## 5. Path B：浏览器预览 fallback 链路

## 5.1 目标

验证浏览器预览端不再因为缺少 Electron bridge 直接失效。

说明：

1. 这条路径验证的是“前端可联调”
2. 不是桌面端真实数据链路验证

## 5.2 步骤 1：打开浏览器预览

操作：

1. 在浏览器中打开 `http://127.0.0.1:8192/`

预期结果：

1. 页面正常打开
2. 不出现“Desktop bridge is unavailable”之类报错

失败判定：

1. 页面直接报 bridge 缺失错误
2. 白屏

## 5.3 步骤 2：搜索示例股票

建议使用以下任一代码：

- `600519`
- `000651`
- `601318`

操作：

1. 输入代码
2. 点击“搜索”

预期结果：

1. 出现搜索结果
2. 可以看到“加入自选”

失败判定：

1. 搜索无结果
2. 页面报错

## 5.4 步骤 3：加入自选并进入收益追踪

操作：

1. 点击“加入自选”
2. 切换到“收益追踪”

预期结果：

1. 自选页展示刚加入的股票
2. 无需 Electron bridge 也能显示

失败判定：

1. 加入后列表为空
2. 页面报错

## 5.5 步骤 4：批量进入对比

操作：

1. 至少加入两只示例股票
2. 分别点“选择”
3. 点击“批量进入对比”

预期结果：

1. 对比页能打开
2. 对比数据能显示

失败判定：

1. 按钮不可用
2. 对比页无内容

## 5.6 步骤 5：验证浏览器端本地存储

操作：

1. 打开浏览器 DevTools
2. 查看 `localStorage`
3. 查找 key：`dm:web-watchlist`

预期结果：

1. 能看到该 key
2. 值是当前浏览器预览端自选代码列表

失败判定：

1. 页面已加入自选但 `localStorage` 无记录

## 6. 通过标准

建议按下列标准判断：

### 6.1 桌面端通过

满足以下条件即可判定桌面端本轮通过：

1. 能搜索并加入自选
2. 不再出现 `watchlist:add` 的 `EPERM`
3. 收益追踪页可看到自选
4. 可从自选批量进入对比
5. 可进入详情和回测
6. 重启后自选仍保留

### 6.2 浏览器预览通过

满足以下条件即可判定浏览器预览本轮通过：

1. 不再因为缺少 `window.dividendMonitor` 直接挂掉
2. 可以搜索示例股票
3. 可以加入自选
4. 可以进入收益追踪与对比页

## 7. 当前已知现象

以下现象当前不作为本轮失败标准：

1. 控制台里存在 `antd Card bordered` 废弃警告
2. 控制台里存在 React Router future flag 警告
3. 浏览器预览使用的是 fallback/mock 数据，不等同于桌面端真实数据
